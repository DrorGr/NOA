Build an interactive LED menorah that lights one more candle with each button press

Project Overview

This Arduino project creates an electronic Hanukkah menorah (Hanukkiah) with 9 LEDs representing the 9 candles. Each button press lights one additional LED, and when all are lit, the next press turns them all off.

How It Works

  • First press: Lights the Shamash (center candle)
  • Presses 2-9: Light candles from right to left
  • Final press (when all lit): Turns all LEDs off
  • Button debouncing: Prevents multiple triggers from single press

Key Features

  • 9 LEDs (8 blue + 1 white for Shamash)
  • Single button control with debouncing
  • Sequential lighting pattern (right to left)
  • Reset when all candles are lit
  • Simple breadboard design
‚ö†Ô∏è Important Notes The Shamash (center candle) should ideally be a different color (white/yellow) while the other 8 candles are traditionally blue. The menorah lights from right to left following Jewish tradition.

Step-by-Step Wiring Instructions

1 LED Setup
  • Insert 9 LEDs into the breadboard in a row
  • Long leg (anode, +) faces toward Arduino pins
  • Short leg (cathode, -) faces toward ground rail
  • Place center LED (Shamash) slightly higher if possible
2 Add Resistors
  • Connect one 220Œ© resistor to the cathode (short leg) of each LED
  • Connect the other end of each resistor to the ground rail on breadboard
  • This limits current and protects the LEDs
3 Connect LEDs to Arduino
  • Connect LED anodes (long legs) to Arduino pins 2-10 sequentially
  • Rightmost LED ‚Üí Pin 2
  • Center LED (Shamash) ‚Üí Pin 6
  • Leftmost LED ‚Üí Pin 10
4 Button Setup
  • Place push button on breadboard
  • Connect one side of button to Arduino Pin 12
  • Connect same side to +5V through 10kŒ© resistor (pull-up)
  • Connect other side of button to GND
5 Microphone Module Setup
  • Connect microphone VCC to Arduino 5V
  • Connect microphone GND to Arduino GND
  • Connect microphone Digital OUT (DO) to Arduino Pin 11
  • Adjust sensitivity potentiometer on module for desired sound threshold
  • Test: LED on module should light up when sound detected
6 Power Connections
  • Connect Arduino GND to breadboard ground rail
  • Connect Arduino 5V to breadboard power rail (if using for button)
  • Ensure all grounds are connected
‚ö†Ô∏è LED Polarity Check LEDs only work in one direction! The flat edge of the LED indicates the cathode (negative side). If an LED doesn't light up, check that it's oriented correctly.

Arduino Code

Copy this code into your Arduino IDE and upload it to your board:

/*
 * Arduino Hanukkah Menorah
 * Lights one LED at a time with each button press
 * When all LEDs are lit, next press turns them all off
 */

// LED pins (right to left, with Shamash in center)
const int ledPins[9] = {2, 3, 4, 5, 6, 7, 8, 9, 10};
const int shamashPin = 6; // Center LED (5th in array, index 4)

// Button configuration
const int buttonPin = 12;

// Microphone configuration
const int micPin = 11;

// State variables
int lightCount = 0;           // How many LEDs are currently lit
int lastButtonState = HIGH;   // Previous button state
int buttonState;              // Current button state
int lastMicState = LOW;       // Previous microphone state
int micState;                 // Current microphone state
unsigned long lastDebounceTime = 0;
unsigned long lastMicDebounceTime = 0;
unsigned long debounceDelay = 50; // Debounce time in milliseconds
unsigned long micDebounceDelay = 300; // Longer delay for microphone to prevent multiple triggers

void setup() {
  // Initialize all LED pins as outputs
  for (int i = 0; i < 9; i++) {
    pinMode(ledPins[i], OUTPUT);
    digitalWrite(ledPins[i], LOW);
  }
  
  // Initialize button pin with internal pull-up
  pinMode(buttonPin, INPUT_PULLUP);
  
  // Initialize microphone pin as input
  pinMode(micPin, INPUT);
  
  // Optional: Serial for debugging
  Serial.begin(9600);
  Serial.println("Hanukkah Menorah Ready!");
  Serial.println("Press button or say a word to light candles");
}

void loop() {
  // Read button state
  int reading = digitalRead(buttonPin);
  
  // Check if button state changed (for debouncing)
  if (reading != lastButtonState) {
    lastDebounceTime = millis();
  }
  
  // If enough time has passed, consider it a valid button press
  if ((millis() - lastDebounceTime) > debounceDelay) {
    // If button state has changed
    if (reading != buttonState) {
      buttonState = reading;
      
      // Only trigger on button press (LOW because of INPUT_PULLUP)
      if (buttonState == LOW) {
        triggerNextCandle("Button");
      }
    }
  }
  
  // Save button state for next loop
  lastButtonState = reading;
  
  // Read microphone state
  int micReading = digitalRead(micPin);
  
  // Check if microphone detected sound
  if (micReading != lastMicState) {
    lastMicDebounceTime = millis();
  }
  
  // If enough time has passed, consider it a valid sound detection
  if ((millis() - lastMicDebounceTime) > micDebounceDelay) {
    // If microphone state has changed
    if (micReading != micState) {
      micState = micReading;
      
      // Trigger on sound detection (HIGH signal from module)
      if (micState == HIGH) {
        triggerNextCandle("Voice");
      }
    }
  }
  
  // Save microphone state for next loop
  lastMicState = micReading;
}

void triggerNextCandle(String source) {
  // Increment light count
  lightCount++;
  
  // If we've lit all candles, reset to 0
  if (lightCount > 9) {
    lightCount = 0;
    Serial.println("Resetting - All candles extinguished");
  }
  
  // Update the LEDs
  updateLEDs();
  
  // Print status
  Serial.print(source);
  Serial.print(" triggered! Candles lit: ");
  Serial.println(lightCount);
}

void updateLEDs() {
  // Turn off all LEDs first
  for (int i = 0; i < 9; i++) {
    digitalWrite(ledPins[i], LOW);
  }
  
  // If no candles should be lit, we're done
  if (lightCount == 0) {
    return;
  }
  
  // Light the Shamash (always lit first)
  digitalWrite(shamashPin, HIGH);
  
  // Light additional candles from right to left
  // Skip the Shamash position (index 4)
  int candlesToLight = lightCount - 1; // Subtract 1 because Shamash is already lit
  
  // Light candles to the right of Shamash (indices 0-3)
  for (int i = 0; i < 4 && i < candlesToLight; i++) {
    digitalWrite(ledPins[i], HIGH);
  }
  
  // Light candles to the left of Shamash (indices 5-8)
  for (int i = 5; i < 9 && (i - 1) < candlesToLight; i++) {
    digitalWrite(ledPins[i], HIGH);
  }
}

Code Explanation

  • ledPins array: Stores all 9 LED pin numbers
  • shamashPin: The center candle (pin 6) that lights first
  • lightCount: Tracks how many candles should be lit (0-9)
  • Debouncing: Prevents multiple triggers from button bounce
  • micPin: Digital pin for microphone sound detection
  • micDebounceDelay: Longer delay (300ms) to prevent multiple triggers from single sound
  • triggerNextCandle(): Unified function for both button and voice activation
  • updateLEDs(): Lights the appropriate LEDs based on count
  • INPUT_PULLUP: Uses Arduino's internal pull-up resistor for button

Customization Options

  • Change debounceDelay if button is too sensitive/unresponsive
  • Adjust micDebounceDelay (300-500ms) to prevent multiple triggers
  • Tune microphone sensitivity using the potentiometer on the module
  • Add LED animations (fade in/out, flicker effect)
  • Add a buzzer to play sounds on activation
  • Modify lighting order or pattern
  • Use analog microphone input for more precise sound detection
‚ö†Ô∏è Microphone Sensitivity The microphone will trigger on ANY sound above the threshold. Adjust the sensitivity potentiometer on the module so it only responds to intentional sounds (clapping, saying a word, etc.) and not background noise.

/*
 * Arduino Hanukkah Menorah with Edge Impulse Voice Recognition
 * Say your trained word to light candles!
 */

// Include your Edge Impulse library (name depends on your project)

// LED pins
const int ledPins[9] = {2, 3, 4, 5, 6, 7, 8, 9, 10};
const int shamashPin = 6;

// Button configuration
const int buttonPin = 12;

// State variables
int lightCount = 0;
int lastButtonState = HIGH;
unsigned long lastDebounceTime = 0;
unsigned long debounceDelay = 50;

// Audio buffer for Edge Impulse
static signed short sampleBuffer[EI_CLASSIFIER_RAW_SAMPLE_COUNT];
static bool debug_nn = false;

void setup() {
  // Initialize LEDs
  for (int i = 0; i < 9; i++) {
    pinMode(ledPins[i], OUTPUT);
    digitalWrite(ledPins[i], LOW);
  }
  
  pinMode(buttonPin, INPUT_PULLUP);
  
  Serial.begin(115200);
  Serial.println("Hanukkah Menorah with Voice Recognition Ready!");
  
  // Initialize Edge Impulse
  if (microphone_inference_start(EI_CLASSIFIER_RAW_SAMPLE_COUNT) == false) {
    Serial.println("ERR: Failed to start microphone");
  }
}

void loop() {
  // Button handling (same as before)
  int reading = digitalRead(buttonPin);
  
  if (reading != lastButtonState) {
    lastDebounceTime = millis();
  }
  
  if ((millis() - lastDebounceTime) > debounceDelay) {
    if (reading != buttonState) {
      buttonState = reading;
      if (buttonState == LOW) {
        triggerNextCandle("Button");
      }
    }
  }
  lastButtonState = reading;
  
  // Voice recognition handling
  bool m = microphone_inference_record();
  if (!m) {
    Serial.println("ERR: Failed to record audio");
    return;
  }
  
  signal_t signal;
  signal.total_length = EI_CLASSIFIER_RAW_SAMPLE_COUNT;
  signal.get_data = &microphone_audio_signal_get_data;
  
  ei_impulse_result_t result = { 0 };
  
  EI_IMPULSE_ERROR r = run_classifier(&signal, &result, debug_nn);
  if (r != EI_IMPULSE_OK) {
    Serial.println("ERR: Failed to run classifier");
    return;
  }
  
  // Check if your keyword was detected with high confidence
  // Replace "hanukkah" with your label name (index depends on your training)
  for (size_t ix = 0; ix < EI_CLASSIFIER_LABEL_COUNT; ix++) {
    if (String(result.classification[ix].label) == "hanukkah") {
      if (result.classification[ix].value > 0.75) {  // 75% confidence threshold
        triggerNextCandle("Voice");
        delay(500);  // Prevent multiple triggers
      }
    }
  }
}

Troubleshooting Tips

  • Low accuracy (<80%): Collect more samples (aim for 100+ per class)
  • False positives: Add more noise samples, increase confidence threshold
  • Not detecting word: Speak closer to microphone, retrain with varied pronunciations
  • Board not connecting: Check USB cable, try different port, reinstall firmware
  • Compilation errors: Verify library name matches your project name
  • High memory usage: Edge Impulse EON compiler should optimize automatically

Alternative: Voice Recognition Module V3

If you prefer hardware-based recognition without Edge Impulse:

  • Purchase Voice Recognition Module V3 (~$10-15)
  • Connect VCC to 5V, GND to GND, RX to Arduino TX, TX to Arduino RX
  • Use Arduino serial commands to train voice commands
  • Module stores 80 commands on-board
  • Check manufacturer documentation for training procedure
